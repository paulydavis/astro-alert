# Hourly Chart Tab — Clear Dark Sky-Style Color Grid

**Date:** 2026-04-27  
**Status:** Approved

## Overview

Add a new "Chart" tab to the AstroAlert GUI showing a 72-hour hour-by-hour color-coded forecast grid (Clear Dark Sky style) for a selected site. A shared HTML generator powers both the GUI rendering and an optional HTML email format selectable in Settings.

## Architecture & Data

### New module: `chart_html.py`
Owns two responsibilities:
1. **Color-mapping functions** — convert raw parameter values to hex colors. Shared by both the GUI Canvas renderer and the HTML generator.
2. **HTML generator** — produces a self-contained HTML email body with an inline-CSS `<table>` (no external CSS, no images).

Public API:
```python
def build_chart_data(site, hours: int = 72) -> ChartData: ...
def render_html(data: ChartData) -> str: ...
```

### `ChartData` dataclass
One list per parameter, 72 entries each. Single object passed to both the Canvas renderer and the HTML generator.

```python
@dataclass
class ChartData:
    site_name: str
    start_dt: datetime          # UTC, hour-aligned
    cloud: list[int]            # 0–100 %
    seeing: list[float]         # 0–8 scale
    transparency: list[float]   # 0–8 scale
    wind: list[float]           # km/h
    humidity: list[int]         # 0–100 %
    temperature: list[float]    # °C
    precipitation: list[float]  # mm
    moon_pct: list[int]         # illumination %, one per hour (spread from daily value)
    moon_events: dict[int, str] # hour index → "rise" or "set"
    errors: list[str]           # data source failures
```

### Data fetching
- **Weather** (cloud, wind, humidity, temp, precip): `fetch_weather` called once with a 3-day range (today → today+2). One Open-Meteo API call.
- **Seeing / Transparency**: `fetch_seeing` — 7timer covers 7+ days, so all 72 hours are available.
- **Moon**: `get_moon_info` called once per day (3 calls). Illumination % spread across all hours of that day; rise/set times recorded in `moon_events`.

### Threading & caching
Data fetched in a background thread on tab load or site change (same `threading.Thread(daemon=True)` pattern as Dashboard and Forecast tabs). Results cached per site for the session. Switching sites clears cache and re-fetches.

## UI Layout

### Tab placement
New "  Chart  " tab inserted after "  Forecast  " and before "  Scoring  ".

### Controls row
- Site selector combobox (single-site only, no "All sites")
- "Load Chart" button (Accent style); disabled and shows "Loading…" during fetch
- Error banner (WARN_CLR) appears below controls if any data source failed; lists affected rows

### Chart canvas
`tk.Canvas` with horizontal and vertical scrollbars. Horizontally scrollable to accommodate 72 columns.

**Layout dimensions:**
- Row label column: 120px wide
- Cell size: 18px wide × 28px tall
- Total chart width: 120 + (72 × 18) = 1416px
- Top rows: day headers (spanning 24 columns each) + hour labels (0–23 repeating)

**Rows (top to bottom):**

| Row | Data | Color scale |
|-----|------|-------------|
| Cloud Cover | 0–100% | Dark blue (0%) → white (100%) |
| Seeing | 0–8 | Dark blue (8) → white (1) |
| Transparency | 0–8 | Dark blue (8) → white (1) |
| Wind | km/h | Dark blue (0) → orange (20) → red (30+) |
| Humidity | 0–100% | Dark blue (<40%) → white (>90%) |
| Temperature | °C | Informational gradient; blue (cold) → neutral (mild) |
| Precipitation | mm | White (0mm) → dark red (≥1mm) |
| Moon | % | Gray cells showing illumination %; ▲ = moonrise hour, ▼ = moonset hour |

**Hover tooltip:** Hovering any cell shows a small tooltip with the raw value (e.g. "Cloud: 42%", "Seeing: 5/8", "Wind: 14 km/h").

## Email & Settings

### Settings tab addition
New "Email Format" section below existing SMTP fields. Two radio buttons:
- **Plain text** — current behavior (default)
- **HTML with chart** — sends the color-coded chart table + go/no-go summary

Selection saved to `.env` as `EMAIL_FORMAT=plain` or `EMAIL_FORMAT=html`.

### HTML email body
Generated by `chart_html.py:render_html(data)`. Contains:
1. Header: site name + date range
2. Color-coded `<table>` with inline `style` on each `<td>` — no external CSS, no images, compatible with common email clients
3. Existing go/no-go summary text below the table

### Notifier integration
`smtp_notifier.py` reads `EMAIL_FORMAT` from `.env`. If `html`, calls `build_chart_data` + `render_html` and sends a `multipart/alternative` MIME message with both plain-text fallback and HTML parts. If `plain` (or key absent), current behavior unchanged.

## Error Handling

| Condition | Behaviour |
|-----------|-----------|
| Weather API failure | Affected cells render dark gray with "?" tooltip; error listed in banner |
| Seeing/transparency failure | Affected row cells render dark gray; error listed in banner |
| Partial data (some hours missing) | Missing hours rendered dark gray; available hours rendered normally |
| HTML render failure during email send | Fall back to plain text; log warning; alert still sends |
| No sites configured | Tab body shows "No sites configured — add one in the Sites tab." |

A partial chart is always shown rather than a blank one.
